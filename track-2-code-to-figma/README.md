# Track 2: Del Código a Figma (Code-to-Design)

En este proyecto exploramos cómo un Design System existente en el código (por ejemplo, reutilizando componentes del Track 1) puede ser importado o sincronizado hacia Figma, para que sirva de base a los diseñadores y no al revés.

## Objetivo
- Manejar el Repositorio de Código como la "Fuente Única de Verdad".
- Exportar o sincronizar este código en archivos o componentes en Figma, logrando una representación visual fiel de la realidad técnica.

## Herramientas a Explorar
- Scripts de sincronización utilizando la **Figma REST API**.
- Plugins como `Storybook to Figma` o plugins de captura `html-to-figma`.

## Siguientes Pasos (Sugeridos)
- [ ] Proveer al menos 1 o 2 componentes ya maquetados en código.
- [ ] Investigar un plugin o script base para convertir este HTML/CSS en Frames de Figma.
